# Student Enrollment Placement Analysis
Institutional BI &amp; Student Success Analytics: An end-to-end Tableau project integrating 4 relational data tables (10,000+ records). Features interactive dashboards tracking placement ratios, revenue realization by fee status, and regional enrollment trends for a professional training institute.

# 🎓 Institutional Growth & Student Lifecycle Analytics
## 📌 Project Overview
This project provides a comprehensive, data-driven analysis of the student lifecycle and business operations for a professional training institution. By integrating **four relational data sources**, the project transforms **10,000+ records** into actionable insights regarding student success, financial health, and regional growth.

## 📊 Business Objectives
* **Revenue Tracking:** Identify which courses and branches drive the highest financial returns.
* **Placement Success:** Correlate academic backgrounds (B.E, M.Tech, etc.) with job placement rates.
* **Financial Oversight:** Monitor fee collection efficiency (Paid vs. Unpaid vs. Partial).
* **Operational Trends:** Compare Online vs. Offline and Weekday vs. Weekend enrollment volumes.

---

## 🏗️ Data Architecture & Relational Model
The project is built on a robust relational structure joined within Tableau using `Student ID`, `Course ID`, and `Branch ID`.

* **Student Master:** Demographic profiles, degrees, and professional experience.
* **Enrollment Log:** The transactional core tracking class modes and placement outcomes.
* **Course Directory:** Course-specific metadata, tech stacks, and fee structures.
* **Branch Directory:** Geographic performance data for Chennai and Bangalore hubs.

---

## 🛠️ Tech Stack & Skills
- **Tool:** Tableau Desktop
- **Modeling:** Relational Joins (Inner/Left), Schema Mapping.
- **Analytics:** KPI Development, Advanced Calculated Fields, Geographic Mapping.
- **Data Prep:** Handling null values, inconsistent date formats, and financial status cleaning.

---

## 🔍 Key Features & Insights

### 1. Placement Analytics
Visualizes the "Placed" vs. "Not Placed" ratio across various degrees. This allows the institution to identify which student segments require additional career support.

### 2. Financial Monitoring
A dedicated view for fee status tracking. This enables the finance team to target "Unpaid" or "Partially Paid" segments to maximize revenue realization.

### 3. Geographic Performance
Mapping branch-level data (e.g., Velachery vs. Marathahalli) to determine regional demand for high-value courses like **Data Science** and **Full Stack Development**.
